# Lumina v0.5c — свайп трекпадом (горизонтальная навигация) — план реализации

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Двухпальцевое горизонтальное смахивание на тачпаде в зоне фото листает каталог
вперёд/назад (по одному кадру за жест), не мешая зуму (вертикаль) и скроллу карусели.

**Architecture:** Чистая функция-аккумулятор `input::on_trackpad_pan` (накопление `dx`, порог →
шаг, разделение по доминантной оси) юнит-тестируется без GPU. Обёртка в `app.rs::MouseWheel`
вызывает её для `PixelDelta` в зоне viewer и клампит результат до ±1 шага навигации.

**Tech Stack:** Rust, winit (`MouseScrollDelta::PixelDelta`), существующий `input`/`navigate`.
Новых зависимостей нет.

```rust
/// Порог накопленного горизонтального смещения тачпада (логические px) на один шаг навигации.
pub const TRACKPAD_NAV_STEP_PX: f32 = 80.0;

/// Накопить горизонтальное смещение тачпада и решить, листать ли.
/// `accum` — аккумулятор (px), мутируется. Возвращает число шагов навигации со знаком:
/// >0 — next N раз, <0 — prev |N| раз, 0 — пока недостаточно.
/// Жест считается горизонтальным только при |dx| >= |dy| (иначе вертикальная прокрутка → сброс).
/// Знак: смахивание влево (dx<0) → next (+1); вправо (dx>0) → prev (−1) — как лента фото.
pub fn on_trackpad_pan(accum: &mut f32, dx: f32, dy: f32, step_px: f32) -> i32 {
    if step_px <= 0.0 || dx.abs() < dy.abs() {
        *accum = 0.0;
        return 0;
    }
    *accum += dx;
    let steps = (*accum / step_px).trunc() as i32;
    if steps != 0 {
        *accum -= steps as f32 * step_px;
    }
    -steps // dx<0 → steps<0 → next(+1); dx>0 → steps>0 → prev(−1)
}
```

- [ ] **Step 2: Обработать горизонтальный `PixelDelta` в viewer-зоне**

В `src/app.rs`, в `WindowEvent::MouseWheel { delta, .. }`, найти финальную ветку зума (где сейчас
`} else if let Some(_win) = win { … on_wheel … }` — навигационный/зум-путь для зоны viewer, НЕ
карусели и НЕ popup). В начало этой ветки (перед вычислением `cursor_v`/`on_wheel`) вставить
перехват горизонтального жеста:
```rust
                } else if let Some(_win) = win {
                    // Тачпад: горизонтально-доминирующий PixelDelta → пошаговая навигация.
                    if let winit::event::MouseScrollDelta::PixelDelta(p) = delta {
                        let dx = p.x as f32;
                        let dy = p.y as f32;
                        let steps = crate::input::on_trackpad_pan(
                            &mut self.state.trackpad_accum, dx, dy, crate::input::TRACKPAD_NAV_STEP_PX);
                        if steps != 0 {
                            // максимум один шаг за событие (инерционный бросок не пролистывает пачку)
                            self.navigate(if steps > 0 { 1 } else { -1 });
                            return;
                        }
                        // горизонтальный жест без полного шага — поглощаем (не зумим)
                        if dx.abs() >= dy.abs() && dx != 0.0 {
                            return;
                        }
                    }
                    // зум как прежде (вертикаль / мышиное колесо)
                    let vw = self.renderer.as_ref().map(|r| r.viewer_size()).unwrap_or_default();
```
> ВАЖНО: это вставка В НАЧАЛО существующей ветки `else if let Some(_win) = win {`. Строки ниже
> (`let cursor_v = …; let out = crate::input::on_wheel(…); …`) остаются без изменений — продолжают
> прежний зум. Не дублировать `let vw = …`, если он уже есть строкой ниже: вставить перехват ДО
> существующего `let vw`, а сам `let vw` оставить один раз (убрать добавленную последнюю строку, если
> приводит к двойному объявлению — она показана лишь как якорь места).
