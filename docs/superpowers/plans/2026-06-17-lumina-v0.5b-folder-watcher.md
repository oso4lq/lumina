# Lumina v0.5b — наблюдатель папки (folder watcher) — план реализации

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Каталог открытой папки автоматически обновляется при добавлении/удалении/переименовании
файлов в ней, сохраняя текущий кадр по пути.

**Architecture:** Чистое ядро согласования (`relocate` + `FolderCatalog::refresh`) тестируется без ФС.
Тонкая обёртка `watcher` мостит `notify` (дебаунс) в winit-event-loop через `EventLoopProxy` как
`UserEvent::FolderChanged`; `app.rs` на это событие пере-сканирует папку и обновляет UI.

**Tech Stack:** Rust, крейты `notify` + `notify-debouncer-mini` (ФС-наблюдение на Windows через
`ReadDirectoryChangesW`), существующий `EventLoopProxy<UserEvent>`.

```rust
/// Индекс курсора в новом списке после пересбора: позиция файла с тем же именем,
/// что у прежнего текущего; если файл исчез — последний валидный индекс (или 0 для пустого).
pub fn relocate(files: &[PathBuf], prev_current: &Path) -> usize {
    if files.is_empty() {
        return 0;
    }
    let prev_name = prev_current.file_name();
    files
        .iter()
        .position(|p| p.file_name() == prev_name)
        .unwrap_or(files.len() - 1)
}
```

```rust
    /// Чистое ядро пересбора: отфильтрованный список → отсортированный + индекс курсора
    /// по прежнему пути. Без обращения к ФС (тестируемо).
    fn rebuild_core(mut files: Vec<PathBuf>, prev_current: &Path) -> (Vec<PathBuf>, usize) {
        files.sort_by(|a, b| {
            let an = a.file_name().and_then(|s| s.to_str()).unwrap_or("");
            let bn = b.file_name().and_then(|s| s.to_str()).unwrap_or("");
            natord::compare(an, bn)
        });
        let current = relocate(&files, prev_current);
        (files, current)
    }

    /// Пере-сканировать папку (повторное чтение ФС), сохранив текущий файл по пути.
    /// Возвращает true, если набор файлов или индекс изменились.
    pub fn refresh(&mut self) -> std::io::Result<bool> {
        let prev_current = self.files.get(self.current).cloned().unwrap_or_default();
        let dir = prev_current.parent()
            .or_else(|| self.files.first().and_then(|p| p.parent()))
            .map(|p| p.to_path_buf())
            .unwrap_or_else(|| std::path::PathBuf::from("."));
        let scanned: Vec<PathBuf> = std::fs::read_dir(&dir)?
            .filter_map(|e| e.ok().map(|e| e.path()))
            .filter(|p| p.is_file() && supported(&ext_lower(p)))
            .collect();
        let (files, current) = Self::rebuild_core(scanned, &prev_current);
        let changed = files != self.files || current != self.current;
        self.files = files;
        self.current = current;
        Ok(changed)
    }
```
> `rebuild_core` использует уже импортированные в модуле `natord` и `relocate`. `supported`/`ext_lower`
> уже импортированы вверху файла.

```rust
//! Тонкая обёртка: наблюдение за папкой через notify (+ дебаунс) с прокидыванием
//! факта изменения в winit-event-loop как UserEvent::FolderChanged. Логики согласования нет.
use crate::app::UserEvent;
use notify_debouncer_mini::{new_debouncer, DebounceEventResult, Debouncer};
use notify::{RecommendedWatcher, RecursiveMode};
use std::path::Path;
use std::time::Duration;
use winit::event_loop::EventLoopProxy;

/// Хранит дебаунсер; его Drop останавливает поток наблюдения.
pub struct FolderWatcher {
    _debouncer: Debouncer<RecommendedWatcher>,
}

impl FolderWatcher {
    /// Начать нерекурсивное наблюдение за `dir`. Любое пакетированное изменение →
    /// один UserEvent::FolderChanged. None при ошибке (наблюдение просто не активируется).
    pub fn watch(dir: &Path, proxy: EventLoopProxy<UserEvent>) -> Option<FolderWatcher> {
        let mut debouncer = new_debouncer(
            Duration::from_millis(300),
            move |res: DebounceEventResult| {
                if res.is_ok() {
                    let _ = proxy.send_event(UserEvent::FolderChanged);
                }
            },
        )
        .ok()?;
        debouncer.watcher().watch(dir, RecursiveMode::NonRecursive).ok()?;
        Some(FolderWatcher { _debouncer: debouncer })
    }
}
```
> Если API `notify-debouncer-mini` в установленной версии отличается (имя `new_debouncer`,
> `DebounceEventResult`, метод `.watcher()`), сверить с `cargo doc -p notify-debouncer-mini --open`
> и адаптировать ВЫЗОВ, сохранив контракт `watch(dir, proxy) -> Option<FolderWatcher>` и отправку
> единственного `UserEvent::FolderChanged` на батч.
