# Lumina v0.4b (часть 2) — редактирование и запись EXIF — дизайн

**Дата:** 2026-06-16
**Фаза:** v0.4b (часть 2 — запись тегов)
**Ветка разработки:** `v0.4b-exif`
**Базовый дизайн:** `docs/superpowers/specs/2026-06-16-lumina-v0.4b-exif-design.md` (общий read/write)
**Часть 1 (чтение) — готова и влита в main.**

## Цель

Дать редактирование и запись тегов в уже существующем EXIF popup: инлайн-правка значений
записываемых групп, удаление тегов и «Удалить всё GPS», буферизация правок с явным
сохранением через exiftool (in-place + `_original`, для RAW — XMP sidecar). Переиспользуется
вся текстовая инфраструктура части 1 (TextEdit, каретка, выделение, фокус, клиппинг).

## Решения брейншторма (часть 2)

| Вопрос | Решение |
|---|---|
| Объём редактирования | Редактируемы **только записываемые группы** EXIF/XMP/IPTC/GPS; File/Composite/ExifTool/ICC_Profile — read-only (без ✎/✕) |
| Модель правки | Инлайн по ✎/клику; активная ячейка = поле TextEdit (каретка/выделение/фокус/клиппинг части 1) |
| Сохранение | Футер **всегда** виден внизу карточки: `[Сохранить]` (неактивна без правок) `[Отменить всё]`. Не нужно скроллить до низа |
| Буфер обмена | Крейт `arboard`: Ctrl+C / Ctrl+V / Ctrl+X в активном поле (поиск или редактор значения) |
| Закрытие с правками | Инлайн-подтверждение в футере: «Несохранённые изменения — Сохранить / Отменить / Продолжить» |
| Бандл exiftool | Перенесён в v0.5 (вместе с инсталлятором). Путь резолвится как в части 1 (рядом с exe / `assets/bin` / PATH) |

## Архитектура (слои)

Сохраняется принцип проекта: «чистое ядро» юнит-тестируется, subprocess/clipboard —
тонкие обёртки без логики. Часть 2 **расширяет** существующие модули части 1, новых
GPU-пайплайнов нет — popup рисуется существующим UI-конвейером (только новые `DrawCmd`).

### `src/exif/tags.rs` (чистое ядро — расширение)

- `const WRITABLE_GROUPS: &[&str] = &["EXIF", "XMP", "IPTC", "GPS"];`
- `fn is_editable(group: &str) -> bool` — `WRITABLE_GROUPS.contains(&group)`.
- `enum TagEdit { Set { group, tag, value }, Delete { group, tag }, DeleteAllGps }`.
- `fn edits_to_args(edits: &[TagEdit]) -> Vec<String>`:
  - `Set` → `-{group}:{tag}={value}`
  - `Delete` → `-{group}:{tag}=`
  - `DeleteAllGps` → `-gps:all=`
  - Порядок детерминированный (стабильный для тестов).
- **Юнит-тесты:** `is_editable` (EXIF/GPS true; File/Composite false); `edits_to_args` для каждого
  варианта и их комбинации.

### `src/exif/write.rs` (обёртка — новый)

- `fn write_edits(path: &Path, edits: &[TagEdit]) -> Result<()>`:
  - `args = edits_to_args(edits)`;
  - **RAW** (классификация — та же, что у роутера `decoder`, напр. `RawDecoder::supports(ext)`):
    sidecar — `exiftool <args> -o %d%f.xmp -- <file>` (создаёт/обновляет `name.xmp`, RAW не трогаем);
  - **остальные** (JPEG/TIFF/PNG/WebP) — in-place; exiftool по умолчанию делает `name.ext_original`;
  - разделитель `--` перед путём (как в `read.rs`); непустой ненулевой выход / stderr → `LuminaError::Exif`.
- `exiftool_path()` — переиспользуется из части 1 (`read.rs`), вынести при необходимости в общее место.
- **Интеграционные тесты `#[ignore]`** (нужен exiftool): запись `Set`/`Delete` на копии JPEG
  (проверить значение и `_original`), запись на RAW-образце (проверить появление `.xmp`).

### `src/ui/textedit.rs` (часть 1 — без изменений API)

Уже содержит `insert_str`/`backspace`/`delete`/`move_*`/`select_all`/`selected_text`/`cut`/
`set_text`/`clear`. `paste(s)` = `insert_str(s)` (отдельный метод не нужен). Clipboard читает/пишет
`app.rs` через `arboard`, дёргая `selected_text()`/`cut()`/`insert_str()`.

### `src/ui/{theme,layout,hit,scene}` — popup (расширение)

- **theme:** цвета — акцент кнопки Save, «опасное» действие (delete/✕), маркер изменённой строки,
  фон/высота футера, приглушённый цвет неактивной Save. Размеры футера и иконок ✎/✕.
- **layout:** тело popup уменьшается на высоту футера (`footer` — зона внизу карточки, над нижним
  краем). В редактируемой строке — прямоугольники зон `edit` (✎) и `del` (✕) справа. Геометрия
  футера (две кнопки) и зоны подтверждения закрытия.
- **hit:** новые регионы `PopupRowEdit(usize)`, `PopupRowDelete(usize)`, `PopupGpsDeleteAll`,
  `PopupSave`, `PopupCancel`, `PopupConfirmSave`, `PopupConfirmDiscard`, `PopupConfirmCancel`.
  Индекс строки — порядковый среди видимых (отфильтрованных) строк, как при отрисовке.
- **scene:** на редактируемых строках по hover — иконки ✎ и ✕ (Tabler); активная строка рисует
  инлайн-поле (фон + текст + каретка/выделение через те же расчёты, что поиск); pending-правки:
  изменённое значение — акцентным цветом (+ маркер), удалённое — зачёркнуто/приглушённо;
  у заголовка группы **GPS** по hover — действие «удалить всё» (✕); футер с `[Сохранить]`/`[Отменить всё]`
  (Save приглушена, если pending пуст); при запросе закрытия с pending — строка подтверждения
  вместо обычного футера.
- **Юнит-тесты:** футер всегда эмитится; ✎/✕ только на editable-строках и только по hover; pending
  Set/Delete дают ожидаемые маркеры/цвета; Save приглушена без правок; регионы хит-теста.

### `src/app.rs` — диспетчер

Состояние (добавить к существующему popup-стейту):
- `exif_pending: BTreeMap<(String, String), PendingOp>` где `PendingOp = Set(String) | Delete`
  (одна операция на `(group,tag)`, повтор перезаписывает);
- `exif_pending_delete_gps: bool`;
- `exif_editing: Option<(String, String)>` — какой тег редактируется (group, tag);
- `exif_editor: TextEdit` — буфер активного редактора (фокус у него, поиск теряет фокус);
- `exif_confirm_close: bool` — показан ли запрос подтверждения закрытия.

Поведение:
- Клик ✎ (или по значению редактируемой строки) → `exif_editing = Some((g,t))`, `exif_editor`
  ставится текущим значением, фокус → editor (каретка мигает в editor, не в поиске).
- **Enter** в editor → записать в `exif_pending` как `Set(value)` (строка помечается «изменено»),
  закрыть редактор. **Esc** в editor → отменить редактирование строки (pending не трогаем).
  Клик вне editor (по другой строке/телу) → коммит текущего редактора, затем действие.
- Клик ✕ на строке → `exif_pending[(g,t)] = Delete` (строка зачёркнута/приглушена). Повторный ✕/✎
  на той же строке снимает/меняет op.
- «Удалить всё GPS» (на заголовке GPS) → `exif_pending_delete_gps = true` (группа помечается к удалению).
- **Save** (активна при наличии правок) → собрать `Vec<TagEdit>` из pending (+ `DeleteAllGps`) →
  spawn `write::write_edits` на rayon → `UserEvent::ExifSaved { generation, result }`:
  - ok → перечитать теги (`read_tags`), очистить pending/delete_gps, обновить заголовок при смене Model;
  - err → баннер со stderr, pending сохранить (можно поправить и повторить).
- **Отменить всё** → очистить pending/delete_gps/editing.
- Clipboard (arboard): Ctrl+C → `selected_text()` в буфер; Ctrl+X → `cut()` в буфер; Ctrl+V →
  `insert_str(clipboard_text)`. Действует на активное поле (editor, иначе поиск).
- Закрытие (`I`/`Esc`/`✕`/клик-вне) при непустых правках → `exif_confirm_close = true` (футер
  показывает подтверждение): Сохранить (→ Save, затем закрыть), Отменить (→ очистить pending и закрыть),
  Продолжить (→ снять флаг). Без правок — закрытие как сейчас.
- Гейтинг ввода части 1 сохраняется (клавиатура/клики/колесо модала). Фокус: поиск ↔ editor
  (клик в поле поиска фокусирует поиск и закрывает editor-коммитом).

## Поток данных (Save)

1. Правки копятся в `exif_pending` (+ `exif_pending_delete_gps`). Save активна, пока есть хоть одна.
2. Save → `edits_to_args` → `write_edits` на rayon → `ExifSaved`.
3. ok → `read_tags` заново (свежие значения, удалённые исчезли) → очистить pending → обновить заголовок.
4. err → баннер stderr; pending не теряется.
5. Устаревшие async-результаты отсекаются по `exif_generation` (как чтение/декод).

## Обработка ошибок и безопасность

- exiftool недоступен → баннер (как в чтении).
- Невалидное значение → exiftool stderr → баннер; pending сохраняется.
- JPEG/TIFF/PNG/WebP — in-place + `name.ext_original` (оригинал восстановим). RAW — только `.xmp`.
- Запись exiftool атомарна (temp + rename); при ошибке частичных правок не остаётся.

## Зависимости

- `arboard` (буфер обмена, кроссплатформенный, pure-Rust обёртка над OS clipboard).
- Новых нативных зависимостей сборки нет. exiftool — внешний (бандл в v0.5).

## Тестирование

- **Юнит:** `tags` (`is_editable`, `edits_to_args`); popup `layout`/`hit`/`scene` (футер, ✎/✕ на
  editable-строках, pending-маркеры, Save-приглушение, регионы, подтверждение закрытия).
- **Интеграция `#[ignore]`** (нужен exiftool): `write_edits` Set/Delete на копии JPEG (значение +
  `_original`); запись на RAW → появление `.xmp`; round-trip read→edit→write→read.
- Ручная GUI-приёмка: правка значения, удаление тега, «Удалить всё GPS», Save, перечитывание,
  заголовок при смене Model, подтверждение при закрытии, clipboard, read-only нередактируемых групп.

## Декомпозиция плана (под-шаги)

1. `tags`: `is_editable` + `TagEdit` + `edits_to_args` (юнит-тесты).
2. `write.rs`: `write_edits` (in-place / RAW sidecar) + `LuminaError::Exif` (интеграц. `#[ignore]`).
3. theme/layout/hit/scene: футер, ✎/✕, инлайн-редактор, pending-маркеры, GPS-delete-all,
   подтверждение закрытия (юнит-тесты геометрии/регионов/эмиссии).
4. `app.rs`: pending-стейт, инлайн-редактирование (Enter/Esc/коммит), delete row, delete-all-GPS,
   Save → `ExifSaved` → перечитывание + заголовок, отмена всё, подтверждение закрытия.
5. Clipboard (`arboard`): Ctrl+C/V/X в активном поле.
6. Ручная приёмка + ROADMAP (v0.4b → 🟢, если приёмка пройдена).
