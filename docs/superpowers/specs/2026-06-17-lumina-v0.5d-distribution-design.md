# Lumina — v0.5d: дистрибутив (portable .zip) + иконка — дизайн

**Дата:** 2026-06-17
**Статус:** дизайн утверждён, готов к плану

## Проблема

Lumina нельзя отдать другому человеку как готовый продукт:

1. **Нет иконки.** Окно и exe используют дефолтную иконку winit/Windows; в taskbar/проводнике/Alt+Tab
   приложение безлико.
2. **Зависит от внешнего окружения.** EXIF-функции требуют `exiftool` в `PATH`; HEIC — нативных
   `heif.dll`/`libde265.dll`/`libx265.dll` в `PATH` или рядом с exe. У конечного пользователя их нет —
   приложение либо не запускается, либо EXIF/HEIC молча не работают.
3. **Нет процедуры сборки дистрибутива.** Чтобы передать сборку, надо вручную собирать exe и
   догадываться, какие файлы положить рядом.

## Решение

Целимся в **portable .zip** (по решению: инсталлятор + ассоциации в реестре — отдельным последующим
циклом). Дистрибутив — самодостаточная папка: `lumina.exe` + все нативные DLL + standalone `exiftool`
(+`exiftool_files`) + иконка, встроенная в exe. Запуск двойным кликом, без установки и без `PATH`.
Иконка решается в этом же цикле (часть «сделать продукт»).

### Часть 1 — Иконка (мастер `app-icon.png`, 1254×1254 RGBA)

Один мастер → два представления, генерируются dev-бинарём (паттерн как `make_fixtures`):

- **`src/bin/make_icon.rs`** (или `[[bin]]` как у `make_fixtures`): читает `app-icon.png`, ресайзит
  до набора размеров и пишет:
  - **`assets/icon/lumina.ico`** — многоразмерный ICO (16/24/32/48/64/128/256 px; 256 — PNG-сжатый
    внутри ICO) через крейт `ico`.
  - **`assets/icon/lumina-256.png`** — 256×256 PNG для иконки окна в рантайме.
  Оба артефакта **коммитятся** (как и фикстуры), чтобы обычная сборка не требовала генерации.

- **Иконка exe (taskbar/проводник/Alt+Tab):** встраивается ресурсом при сборке. `build.rs` через крейт
  `embed-resource` компилирует `assets/icon/lumina.rc` (`IDI_ICON1 ICON "lumina.ico"`). Только под
  Windows (`cfg(windows)` / target-check в build.rs).

- **Иконка окна (рантайм):** в `app.rs::resumed` до/при создании окна — `include_bytes!` для
  `assets/icon/lumina-256.png`, декод через `image` → `winit::window::Icon::from_rgba(rgba, 256, 256)`
  → `WindowAttributes::with_window_icon(Some(icon))`. (Встроенного ресурса достаточно для taskbar, но
  winit окну иконку задают отдельно — оба нужны.)

### Часть 2 — Бандл нативных зависимостей и exiftool

Дистрибутив-папка рядом с `lumina.exe`:
- **HEIC DLL:** `heif.dll`, `libde265.dll`, `libx265.dll` (из `vcpkg/installed/x64-windows/bin`).
  Кладутся рядом с exe — Windows ищет DLL в каталоге exe.
- **exiftool standalone:** `exiftool.exe` + папка `exiftool_files/` (официальная Windows-сборка
  exiftool с [exiftool.org](https://exiftool.org), «Windows Executable»). `exif::read::exiftool_path()`
  **уже** ищет `exiftool.exe` рядом с exe — менять код не нужно. Standalone exe требует `exiftool_files/`
  в том же каталоге.
- Лицензии/README: короткий `README.txt` (как запускать) и файлы лицензий бандлимых компонент
  (exiftool — Perl Artistic/GPL; libheif — LGPL; libde265/x265 — отметить). Их наличие в zip — нужная
  гигиена распространения.

### Часть 3 — Скрипт упаковки

**`packaging/package.ps1`** (PowerShell): детерминированная сборка дистрибутива.
1. `cargo build --release`.
2. Создать staging-папку `dist/Lumina/`.
3. Скопировать: `target/release/lumina.exe`; три DLL из `$env:VCPKG_ROOT/installed/x64-windows/bin`;
   `exiftool.exe` + `exiftool_files/` (из заданного пути к standalone exiftool — параметр скрипта);
   `README.txt` и лицензии.
4. Запаковать `dist/Lumina/` в `dist/Lumina-<версия>-x64.zip` (`Compress-Archive`).
Скрипт идемпотентен (чистит staging) и параметризован путями к vcpkg и exiftool. Версия берётся из
`Cargo.toml`.

### Границы (осознанно)

- **Только portable .zip.** Инсталлятор (Inno Setup), ассоциации файлов в реестре, запись в
  «Установка и удаление программ» — следующий цикл (v0.5e). `exiftool_path()` уже совместим с обоими
  сценариями (рядом с exe / `assets/bin`).
- **Только x64.** Единственная целевая архитектура проекта.
- **Точный набор DLL** подтверждается на чистой машине (без vcpkg в `PATH`): запустить из распакованного
  zip и открыть HEIC. Если потребуются транзитивные DLL — добавить в список скрипта (на текущей машине
  vcpkg bin содержит ровно эти три).
- **Подпись кода** (Authenticode) — вне объёма (нет сертификата); SmartScreen покажет предупреждение —
  это ожидаемо для неподписанного portable.

## Тестирование

- **Иконка:** `cargo run --bin make_icon` создаёт `assets/icon/lumina.ico` и `lumina-256.png`
  (проверить размеры/наличие). Сборка с `embed-resource` проходит; `cargo run` — у окна и в taskbar
  кастомная иконка.
- **make_icon** покрыть лёгким юнит-тестом чистой части (выбор размеров) — если логика тривиальна
  (фиксированный список), тест необязателен; достаточно ручной проверки артефактов.
- **Дистрибутив (ручная приёмка, ключевая):** на машине **без** vcpkg в `PATH` и **без** установленного
  exiftool распаковать zip → запустить `lumina.exe` → открыть JPG (работает), HEIC (декодируется, DLL
  подхватились), EXIF popup (читается, exiftool рядом), правка тега (пишется). Иконка корректна.
- Регресс юнит-тестов не затрагивается (изменения — build.rs/ресурс/скрипт/dev-бинарь).
