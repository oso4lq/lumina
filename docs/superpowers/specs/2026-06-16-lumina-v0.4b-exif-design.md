# Lumina v0.4b — EXIF чтение/запись — дизайн

**Дата:** 2026-06-16
**Фаза:** v0.4b (EXIF popup + запись тегов)
**Ветка разработки:** `v0.4b-exif`

## Цель

EXIF popup поверх viewer (wgpu-overlay): **полный браузер тегов** на чтение и
**редактирование любого тега** с записью в файл. Модель камеры — в заголовке окна.
Запись через бандлёный `exiftool` (subprocess), in-place с `_original`-бэкапом;
для RAW — XMP sidecar.

## Решения брейншторма

| Вопрос | Решение |
|---|---|
| Объём фазы | Полное чтение **и** запись (не только чтение) |
| Механизм записи | Бандл `exiftool.exe` + вызов как subprocess |
| Модель редактирования | Полный браузер тегов (все теги exiftool, любой редактируем) |
| Текстовый ввод | Стандартный однострочный (каретка ←/→/Home/End, выделение, буфер обмена, Unicode; без IME-композиции) |
| Бэкап при записи | In-place + `_original` (exiftool по умолчанию); RAW → `.xmp` sidecar |
| Раскладка popup | Сгруппировано по категориям exiftool + поле поиска/фильтра |

## Архитектура (слои)

Сохраняется принцип проекта: «чистое ядро» юнит-тестируется, FFI/subprocess —
тонкие обёртки без логики.

### `src/exif/`

- **`mod.rs`** (существует) — `Orientation` read/apply, без изменений.
- **`tags.rs`** (новое, чистое ядро) — модель данных и преобразования:
  - `ExifTags` — упорядоченный список групп (`EXIF`, `GPS`, `XMP`, `IPTC`, `File`,
    `Composite`, …), внутри — `[(tag, value)]` в порядке вывода exiftool.
  - `parse(json: &str) -> ExifTags` — разбор `exiftool -json -G` (массив с одним объектом;
    ключи вида `"Group:Tag"`, значения — строки/числа приводятся к `String`).
  - `TagEdit` — операция над тегом: `Set(group, tag, value)` / `Delete(group, tag)` /
    `DeleteAllGps`.
  - `edits_to_args(edits: &[TagEdit]) -> Vec<String>` — формирование аргументов exiftool
    (`-Group:Tag=value`, удаление `-Group:Tag=`, `-gps:all=`).
  - **Юнит-тесты:** парсинг JSON-фикстуры, формирование аргументов из набора правок.
- **`read.rs`** (обёртка) — `read_tags(path) -> Result<ExifTags>`: запустить
  `exiftool -json -G -struct <file>`, отдать stdout в `tags::parse`. Ошибка запуска/непустой
  stderr → `LuminaError`.
- **`write.rs`** (обёртка) — `write_edits(path, edits) -> Result<()>`:
  - собрать аргументы через `tags::edits_to_args`;
  - для редактируемых форматов (JPEG/TIFF/PNG/WebP) — in-place (exiftool сам делает
    `name.ext_original`);
  - для RAW — писать в sidecar: `exiftool <args> -o %d%f.xmp <file>` (создаёт/обновляет
    `name.xmp`, оригинал RAW не трогаем);
  - вернуть `Result` с текстом stderr exiftool при ошибке.

### `src/ui/textedit.rs` (новое, чистое ядро)

Состояние однострочного редактора, **без GPU/winit**:

- поля: `buffer: String`, `caret: usize` (в символах), `sel_anchor: Option<usize>`;
- операции: `insert_str`, `backspace`, `delete`, `move_left/right`, `move_home/end`
  (+ вариант с shift = расширение выделения), `select_all`, `selected_text`, `cut`,
  `paste(&str)`, `take`/`set`;
- корректная работа по символам (не байтам) для Unicode.
- **Юнит-тесты** на все операции (каретка, выделение, вставка, удаление выделенного).

`app.rs` лишь маршрутизирует события клавиш и буфера обмена в эти методы; рендер
поля — обычные `DrawCmd` (фон + текст + каретка-прямоугольник).

### `src/ui/{theme,layout,hit,scene}` — popup

- **`theme`** — размеры/цвета popup (карточка, строка, футер, акцент кнопки Save, цвет
  опасного действия, фон-затемнение overlay).
- **`layout`** — геометрия popup при заданном размере окна: центр-карточка (ширина/высота
  с клампом под маленькие окна), зона заголовка, поле поиска, скролл-область строк
  (с учётом `scroll` и высоты строки), футер с кнопками. Возвращает прямоугольники.
- **`hit`** — регионы popup: `PopupClose`, `PopupSearch`, `PopupRow(idx)`,
  `PopupRowEdit(idx)`, `PopupRowDelete(idx)`, `PopupDeleteGps`, `PopupSave`, `PopupCancel`,
  плюс «клик вне карточки» → закрытие. Когда popup открыт, hit popup имеет приоритет над
  обычными регионами окна.
- **`scene`** — `DrawCmd`-эмиссия popup: затемнение фона, карточка, заголовок + `✕`,
  поле поиска, заголовки групп, строки (ключ/значение, hover-подсветка, ✎/✕ по hover),
  активное поле редактирования (textedit), баннер ошибки, футер.
- **Юнит-тесты** геометрии/регионов/эмиссии (как для bottom bar и стрелок).

### `src/app.rs` — диспетчер

- Состояние: `exif_popup_open: bool`, `exif_tags: Option<ExifTags>`,
  `exif_pending: BTreeMap<(String,String), PendingOp>` (ключ — `(group, tag)`,
  `PendingOp = Set(String) | Delete`; одна операция на тег, повтор перезаписывает),
  `exif_pending_delete_gps: bool`, `exif_editing: Option<usize>` (индекс строки),
  `exif_textedit: TextEdit`, `exif_scroll: f32`, `exif_filter: String`,
  `exif_error: Option<String>`, `exif_generation` (как у декода — отсечь устаревшее).
- Открытие (`I`) → spawn `read::read_tags` на rayon → `UserEvent::ExifLoaded{generation, result}`.
- Правки буферизуются; **Save** → spawn `write::write_edits` → `UserEvent::ExifSaved{result}`
  → при ok перечитать теги и обновить заголовок (если сменился `Model`); при err — баннер.
- exiftool вызывается только на rayon (event loop не блокируется).

### Бандл exiftool

- `assets/bin/exiftool.exe` (+ сопутствующие файлы дистрибутива Windows) кладётся в репозиторий
  или подтягивается в сборку; путь резолвится относительно текущего exe
  (`std::env::current_exe()`), с фолбэком на `exiftool` из `PATH` для dev-запусков.
- Документировать в `CLAUDE.md`/`ROADMAP.md` (как vcpkg/HEIC).

## Поток данных

1. **Открытие popup** (`I`) → `exif_generation += 1` → spawn `read_tags` → `ExifLoaded` →
   сохранить `exif_tags`, сбросить `exif_pending/filter/scroll/error` → рендер.
2. **Правка**: клик ✎ на строке → `exif_editing = Some(idx)`, `textedit` инициализируется
   текущим значением → ввод → **Enter** коммитит `TagEdit::Set` в `exif_pending`, **Esc**
   отменяет редактирование строки. Клик ✕ на строке → `TagEdit::Delete` в pending
   (строка помечается зачёркнутой/приглушённой). «Удалить всё GPS» → `TagEdit::DeleteAllGps`.
3. **Save** → собрать `Vec<TagEdit>` из `exif_pending` (+ `DeleteAllGps`, если флаг взведён) →
   `edits_to_args` → spawn `write_edits` → `ExifSaved`:
   - ok → перечитать теги (свежие значения, исчезли удалённые), очистить pending,
     обновить заголовок при смене `Model`;
   - err → показать stderr в баннере, pending сохранить (можно поправить и повторить).
4. **Заголовок окна**: при загрузке фото (`load_current`) — дешёвое чтение `Model` через
   `kamadak-exif` (уже зависимость; **не** запускаем exiftool на каждой навигации) →
   `имя · Model · Lumina`. Нет Model (или формат без EXIF, напр. часть HEIC) → `имя · Lumina`.

## Открытие/закрытие и фокус

- `I` — toggle popup; `Esc` — закрыть popup, если открыт, иначе прежнее поведение
  (выход из fullscreen); `✕` и клик вне карточки — закрыть.
- Пока popup открыт: шорткаты трансформаций/навигации/зума **не срабатывают** (фокус у popup);
  колесо мыши скроллит список тегов; ввод символов идёт в активное поле редактирования.
- При закрытии с непустым `exif_pending` — подтверждение «Отменить несохранённые изменения?».

## Запись / безопасность

- JPEG/TIFF/PNG/WebP — in-place, `name.ext_original` рядом (оригинал восстановим). Возможная
  будущая кнопка «очистить бэкапы» — вне scope этой фазы.
- RAW — всегда `name.xmp` sidecar; сам RAW неизменен.
- Запись exiftool атомарна (temp-файл + rename), частичных правок при ошибке не остаётся.

## Обработка ошибок

- exiftool не найден/не запустился → баннер «exiftool недоступен» (и чтение, и запись через него).
- Невалидное значение тега → exiftool пишет в stderr → баннер; pending не теряется.
- Устаревшие async-результаты отсекаются по `exif_generation` (как у декода миниатюр/фото).

## Тестирование

- **Чистое ядро (юнит):** `tags.rs` (парс JSON-фикстуры, дифф→аргументы),
  `textedit.rs` (каретка/выделение/вставка/удаление), popup `layout`/`hit`/`scene`
  (геометрия, регионы, эмиссия команд, приоритет над окном).
- **Интеграция (`#[ignore]`):** `read.rs`/`write.rs` на реальном файле (как HEIC/RAW-тесты) —
  чтение тегов, запись Set/Delete/DeleteAllGps, проверка `_original`/`.xmp`.

## Декомпозиция плана (под-шаги)

1. `textedit` ядро + рендер однострочного поля (каретка/выделение).
2. exif read (`exiftool -json`) + модель `tags` + popup на **чтение**
   (theme/layout/hit/scene) + `Model` в заголовке.
3. Редактирование: pending-правки, инлайн ✎, delete row, «Удалить всё GPS»; интеграция textedit.
4. exif write (`exiftool` in-place+backup, RAW sidecar) + Save + ошибки + перечитывание.
5. Полировка: поиск-фильтр, скролл popup, подтверждение несохранённого, бандл exiftool + резолв пути.

## Вне scope (на будущее)

- Кнопка очистки `_original`-бэкапов.
- IME-композиция (CJK-ввод) в текстовом поле.
- Сортировки каталога по EXIF-дате (v0.6, см. SPEC §сортировка).
- Запись Orientation как отдельная операция «запечь ручной поворот в файл» — покрывается
  общим редактированием тега Orientation; отдельный UX-ярлык — при необходимости позже.
