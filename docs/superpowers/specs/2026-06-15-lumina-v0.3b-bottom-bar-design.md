# Дизайн v0.3b — Bottom bar (divider, мета-панель, карусель миниатюр, кнопки действий)

> Фаза следует за v0.3a (движок UI-примитивов + frameless titlebar). Строится **поверх** чистого
> ядра `src/ui/` (`theme`/`layout`/`hit`/`scene`), `UiPipeline` (SDF-rect) и `TextLayer` (glyphon).
> Спецификация компонентов — `SPEC_lumina.md` §4.3–4.4, дорожная карта — `ROADMAP.md`.

## Цель

Добавить нижнюю панель просмотрщика: горизонтальный **divider** (скрытие/показ панели),
**мета-панель** файла слева, **карусель миниатюр** по центру и **кнопки действий** справа
(fullscreen + EXIF). Ввести шрифт иконок **Tabler** для кнопок действий и реальный
**полноэкранный режим**.

## Решения брейншторма

- **Объём:** вся v0.3b — одной спекой и одним планом (внутри — изолированные модули).
- **Миниатюры:** ленивый декод in-memory с LRU; persistent-кэш (sled) и префетч±2 — отложены в **v0.5**.
- **Иконки действий:** подключаем **Tabler Icons** (.ttf, MIT) через glyphon — как в спеке; кнопки окна
  остаются на Segoe MDL2 (v0.3a).
- **Кнопка EXIF:** рисуется (раскладка финальная по спеке), но **инертна** — реальный EXIF popup и чтение
  тегов приходят в **v0.4**.
- **Fullscreen:** иммерсивный — весь хром (titlebar + divider + bottom bar) скрыт, фото на весь монитор;
  вход/выход по `F`/`F11`/кнопке и `Esc`.
- **Раскладка (подтверждена мокапом):** высоты (логич. px) titlebar 32 · divider 22 · bottom bar ≈ 84
  (миниатюра 64 + отступы 10/10); зоны bottom bar — мета 110 · карусель гибкая · действия 76;
  миниатюры 62×64, скругление 4, активная — белая рамка 1.5px, бейдж формата для RAW.

## Архитектура

Расширяем чистое ядро `ui` (раскладка/hit/сцена остаются юнит-тестируемыми, без GPU). Миниатюры
(изображения) — отдельный асинхронный `ThumbnailStore` + тонкий GPU-слой `ThumbnailLayer`. Логики в
GPU/winit/FFI-обёртках нет.

### Порядок рендера кадра

фото в viewer-viewport → **миниатюры** (textured-quad по их rect'ам) → UI-прямоугольники (фон
bottom bar/divider, hover, активная рамка) → текст (мета, бейджи, ti-глифы, заголовок). Активная
рамка и бейджи ложатся поверх миниатюр.

## Чистое ядро `ui`

### `layout.rs`

`compute(win, scale, bottom_factor, fullscreen)` — всё в физ. px. `bottom_factor ∈ [0,1]` — анимированная
видимость bottom bar (0 — скрыт, 1 — полностью; промежуточные значения во время слайда ~200 мс):

- `fullscreen=true` → весь хром нулевой (`titlebar/divider/bottom_bar` пустые), `viewer` = весь экран.
- Иначе: сверху `titlebar` (32·s). `divider` (22·s) виден **всегда** (это ручка toggle), прижат к верхней
  кромке bottom bar. `bottom_bar` высотой `84·s·bottom_factor`. `viewer` = пространство между titlebar и
  верхом divider.
- `bottom_factor=0` → `bottom_bar` высотой 0, `divider` прижат к низу окна, `viewer` тянется до divider.
- Внутри bottom bar: `meta` (слева 110·s), `btn_exif`/`btn_fullscreen` (справа, зона 76·s), `carousel`
  между ними. При `bottom_factor<1` bottom bar частично за нижней кромкой (слайд); содержимое
  отрисовывается обрезанным viewport'ом.

Чистая `carousel_thumb_rects(carousel, count, scroll, scale) -> Vec<(usize, Rect)>`: x i-й миниатюры =
`carousel.x − scroll + i·(THUMB_W+GAP)·s`, по вертикали — центр `carousel`; включаются только
пересекающие область карусели (видимые).

### `hit.rs`

`Region` += `Divider, Carousel, Thumbnail(usize), ActionFullscreen, ActionExif`.

Приоритет: края resize → кнопки titlebar → caption (v0.3a) → `btn_fullscreen`/`btn_exif` → `divider` →
миниатюра → `carousel` → `None`. Попадание по миниатюре — отдельной чистой функцией
`hit_thumbnail(carousel, count, scroll, scale, cursor) -> Option<usize>` (зависит от scroll, тестируется
изолированно). Fullscreen-инертность реализуется в app/нативном слое, не в `hit`.

### `scene.rs`

```rust
pub struct FileMeta {
    pub format_label: String, // "RAF · RAW"
    pub megapixels: f32,
    pub width: u32,
    pub height: u32,
    pub bytes: u64,
}
// чистая meta_lines(&FileMeta) -> Vec<(String /*label*/, String /*value*/)>
// округление MP до 0.1; гуманизация байт ("38.4 МБ")
```

`UiState` += `bottom_visible: bool` (цель toggle), `bottom_factor: f32` (0..1, анимированная высота/альфа
шеврона), `fullscreen: bool, meta: Option<FileMeta>, thumb_count: usize, active_index: usize, scroll: f32`.

`DrawCmd::Icon` += `font: IconFont` (`WindowMdl2` / `Tabler`).

`build`:
- `fullscreen` → только фон viewer (хрома нет).
- иначе: titlebar (v0.3a) + `divider` (фон, грип, метка «карусель» с альфой `bottom_factor`, шеврон с
  поворотом по `bottom_factor`); если `bottom_factor>0` — фон bottom bar, мета-текст (`meta_lines`), активная
  рамка (Rect поверх активной миниатюры, если её индекс в видимой зоне), бейджи формата (Text над
  RAW-миниатюрами), две `Icon{font: Tabler}` (fullscreen/exif) с hover-подложкой; EXIF тусклее (inert).

## Миниатюры

### `ThumbnailStore` (`src/thumbnail.rs`; чистая логика тестируема, rayon-spawn — из `app.rs`)

- `HashMap<usize, ThumbState>` где `ThumbState = Loading | Ready | Failed`; LRU-список индексов
  (лимит ~256).
- `folder_generation: u64` — инкремент при смене папки; результаты старого поколения игнорируются.
- `request(visible_indices)` — для видимых ± запас, которых нет в карте: ставит `Loading`, спавнит
  `rayon::spawn`: источник миниатюры = `decoder::decode_preview(path)` если `Some` (встроенный JPEG RAW,
  мгновенно), **иначе** `decode_full(path)` (для JPEG/PNG/HEIC, у которых превью нет) → ресайз до 62×64·s
  («cover»-кроп под аспект) → `RGBA8` → `proxy.send_event(UserEvent::Thumbnail{folder_generation, index,
  rgba, w, h})`. (Декод полного кадра для не-RAW миниатюр приемлем в v0.3b — лениво и на rayon; ускорение
  через `image`-thumbnail/sled — в v0.5.)
- Бейдж формата — из расширения файла каталога, не из миниатюры.
- LRU-эвикция: при превышении лимита `remove(index)` (освобождает GPU-текстуру через `ThumbnailLayer`).

### `ThumbnailLayer` (`src/renderer/thumbnail.rs`) + `assets/shaders/thumb.wgsl`

- Текстура на миниатюру: `HashMap<usize, (wgpu::Texture, BindGroup)>`; `set(index, rgba, w, h)` грузит
  `Rgba8UnormSrgb`; `remove(index)`.
- `draw(pass, screen, &[(index, Rect)])`: для каждой видимой готовой миниатюры — квад в экранных px → NDC;
  фрагмент сэмплит текстуру и применяет SDF-скругление (radius 4·s) как альфа-маску. Несколько видимых
  миниатюр = несколько дешёвых draw-вызовов (атлас — позже, v0.5).

### Tabler в `text.rs`

- `assets/fonts/tabler-icons.ttf` (MIT) через `include_bytes!`; в `TextLayer::new` —
  `font_system.db_mut().load_font_data(bytes.to_vec())`; имя семейства — константа (фактическое сверяется
  при сборке).
- `IconFont {WindowMdl2 → "Segoe MDL2 Assets", Tabler → "<tabler family>"}`; `prepare` выбирает семейство
  по `DrawCmd::Icon.font`.
- Кодпоинты `ti-maximize` / `ti-info-circle` — из карты глифов Tabler (PUA), заданы константами, сверяются
  при сборке.

`Renderer::render` получает доп. параметр — список `(index, Rect)` видимых миниатюр (rect'ы из
`layout::carousel_thumb_rects`); рисуются между фото и UI-rect'ами. `set_thumbnail(index, rgba, w, h)`
пробрасывается в `ThumbnailLayer`.

## Интеграция в App

`AppState` += `bottom_visible: bool (цель, default true), bottom_factor: f32 (анимир., default 1.0),
fullscreen: bool, carousel_scroll: f32, thumbs: ThumbnailStore`. `active_index` берётся из `catalog`.

`UserEvent` += `Thumbnail { folder_generation, index, rgba, w, h }` → `user_event`: проверка поколения,
`renderer.set_thumbnail(...)`, `thumbs` помечает `Ready`, `request_redraw`.

Смена папки/фото (`open_file`/`load_current`): строим `FileMeta` (размеры — из декода фото, размер файла —
`fs::metadata`, формат — из ext); `thumbs.folder_generation += 1`; запрашиваем миниатюры для видимого окна
(active ± запас); автоскролл, чтобы активная была видна.

`window_event`:
- `CursorMoved` — hover включает регионы bottom bar.
- `MouseWheel` — маршрутизация по региону под курсором: над `Carousel`/`Thumbnail` → горизонтальный скролл
  карусели (clamp к содержимому), иначе прежний zoom.
- `MouseInput` Pressed (приоритеты): кнопки titlebar → `ActionFullscreen` → `ActionExif` (no-op) →
  `Divider` (инвертировать `bottom_visible` — цель анимации) → `Thumbnail(i)` (навигация на индекс) →
  иначе прежняя логика (double-click/pan).
- `KeyboardInput` — `F`/`F11` → toggle fullscreen; `Esc` → выход из fullscreen.

Fullscreen toggle: `window.set_fullscreen(Some(Borderless(None)))` / `None`; `state.fullscreen`; инсеты
хрома → 0; `#[cfg(windows)] platform::windows::set_fullscreen(bool)`.

Инсеты вида: `Renderer` += `set_bottom_chrome_height(divider+bottom_bar)`; `viewer_size()` = высота между
titlebar и нижним хромом; в fullscreen обе → 0. Курсор zoom-к-точке корректируется на высоту titlebar (как
в v0.3a); скролл/драг карусели на это не влияют.

Анимация toggle (~200 мс): в `tick(dt)` `bottom_factor` ease-out движется к `1.0` если `bottom_visible`
иначе к `0.0` (как zoom-анимация view); высота bottom bar и поворот шеврона/альфа метки берут `bottom_factor`;
пока `bottom_factor` не достиг цели — `request_redraw`. Инсет вида = `divider + 84·s·bottom_factor`.

`RedrawRequested`: `layout::compute(win, scale, bottom_factor, fullscreen)` →
`carousel_thumb_rects` → `scene::build` → запрос недостающих видимых миниатюр →
`renderer.render(view, &cmds, &visible_thumb_rects)`.

## Нативный слой (`platform/windows.rs`)

- Доп. `FULLSCREEN: AtomicBool` + `set_fullscreen(bool)`.
- `WM_NCHITTEST`: если `FULLSCREEN` → сразу `HTCLIENT` (нет caption-drag и resize-краёв). Вне fullscreen —
  прежняя логика (`hit::hit`; divider/bottom bar попадают в `HTCLIENT`, клики ловит winit; нижний 6-px
  resize-край сохраняется приоритетом краёв).
- `WM_NCCALCSIZE` без изменений.

## Обработка ошибок

- Миниатюра не декодировалась → `ThumbState::Failed`, в карусели рисуется плейсхолдер (тёмный rect), без
  падения; `warn`.
- `decode_preview` → `None` → фолбэк (decode_full уменьшенный / плейсхолдер), без блокировки UI.
- Tabler не загрузился/нет глифа → .notdef; `warn`; UI не падает.
- `set_fullscreen` при отсутствии монитора → no-op.
- Рендер миниатюр устойчив к вырожденным rect'ам (гард как в v0.3a — пустой/невидимый rect пропускается).

## Тестирование

**Чистое ядро (юниты, как v0.3a):**
- `layout`: split viewer/divider/bottom при visible/hidden/fullscreen; rect'ы зон и масштаб; позиции и
  видимость миниатюр при скролле; clamp скролла.
- `hit`: divider/кнопки/миниатюра/край-приоритет; `hit_thumbnail` по индексам при скролле.
- `scene`: состав команд при hidden/fullscreen; активная рамка только в видимой зоне; бейдж для RAW; две
  ti-иконки; тусклость EXIF.
- `meta_lines`: округление MP, гуманизация байт.
- `ThumbnailStore`: выбор индексов под запрос, LRU-эвикция, игнор старого поколения, cover-кроп-расчёт.

**GPU/winit/FFI:** сборка + ручная визуальная приёмка — карусель скроллится и грузится лениво, активная
подсвечена, бейджи, divider toggle с анимацией, fullscreen скрывает хром и вход/выход по `F`/`Esc`, кнопки
работают, мета-панель обновляется при смене фото.

- Реальный EXIF popup и чтение тегов — **v0.4** (кнопка в v0.3b инертна).
- Slideshow-тулбар — **v0.6**.
