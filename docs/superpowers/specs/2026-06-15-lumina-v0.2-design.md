# Lumina v0.2 — дизайн (RAW и форматы)

Дата: 2026-06-15
Статус: одобрен к реализации
Базовая спецификация: `SPEC_lumina.md` (v0.2)
Предыдущая фаза: `docs/superpowers/specs/2026-06-15-lumina-v0.1-design.md`

## Цель v0.2

Поддержка RAW и HEIC поверх работающего viewer-ядра v0.1 (§12 спеки):

- [ ] RAW через `rawler` (RAF/NEF/ARW/CR2 и прочее, что тянет rawler) — pure Rust
- [ ] HEIC/HEIF через `libheif-rs`
- [ ] Embedded preview для мгновенного показа
- [ ] Async полное декодирование RAW (базовый develop)

Осознанно отложено (см. «Границы фазы»): префетч ±2 и LRU-кэш текстур (v0.5), `tokio`
(остаёмся на rayon), `libraw-sys` fallback, UI-бейдж формата (v0.3 вместе с titlebar),
точный камерный цвет/film simulation.

## Архитектурный подход

Сохраняем конвейер v0.1 (всё состояние в главном потоке; декод на rayon; готовый RGBA
возвращается через `EventLoopProxy` с монотонным `generation_id`). Расширяем:

1. **Trait `Decoder` получает две стадии** — `decode_preview` и `decode_full`.
2. **Роутер по расширению** выбирает один из трёх декодеров.
3. **Одна rayon-задача на файл** эмитит сначала Preview, затем Full.
4. **Бесшовная подмена preview→full** в `ViewTransform` (сохранение экранного размера).

Отвергнуто: поток стадий через callback/канал (избыточно для двух стадий, хуже тестируется);
вынос превью в отдельную функцию вне trait (превью формат-специфично — принадлежит декодеру).

## Структура модулей

```
src/decoder/
├── mod.rs        # trait Decoder (расширен), DecodedImage, роутер supported()/decoder_for()
├── standard.rs   # как в v0.1; decode_preview → None
├── raw.rs        # RawDecoder (rawler): embedded preview + базовый develop
└── heic.rs       # HeicDecoder (libheif-rs): одностадийный
```

Меняется минимально: `src/app.rs` (двухстадийная диспетчеризация + подмена вида),
`src/catalog/mod.rs` (фильтр по `decoder::supported`), `src/view.rs` (метод подмены),
`Cargo.toml` (зависимости). Прочие модули v0.1 не трогаем.

### Trait Decoder (расширение)

```rust
pub trait Decoder {
    fn supports(ext: &str) -> bool where Self: Sized;

    /// Быстрое встроенное превью. None — у формата превью нет (одна стадия).
    fn decode_preview(&self, path: &Path) -> Result<Option<DecodedImage>>;

    /// Полное декодирование (для RAW — базовый develop).
    fn decode_full(&self, path: &Path) -> Result<DecodedImage>;
}
```

`supports` помечен `where Self: Sized` → не попадает в vtable, поэтому `decode_preview` и
`decode_full` (оба `&self`) делают trait object-safe.

### Роутер

```rust
pub fn supported(ext: &str) -> bool {
    StandardDecoder::supports(ext) || RawDecoder::supports(ext) || HeicDecoder::supports(ext)
}

/// Приоритет: Raw → Heic → Standard (расширения не пересекаются, но порядок фиксирован).
pub fn decoder_for(ext: &str) -> Option<Box<dyn Decoder + Send>> { ... }
```

`catalog` переключается с `StandardDecoder::supports` на `decoder::supported(ext)` — RAW/HEIC
файлы участвуют в скане и navigation. `Box<dyn Decoder + Send>` уезжает в rayon-задачу.

## Поток данных (двухстадийный)

```
1. App получает индекс файла → AppState.generation += 1
2. rayon::spawn:
     ext = ext_lower(path); decoder = decoder_for(ext)
     ├─ decode_preview(path):
     │     Ok(Some(img)) → proxy.send(Decoded { gen, stage: Preview, Ok(img) })
     │     Ok(None) | Err → превью пропускаем (файл не валим)
     └─ decode_full(path) → proxy.send(Decoded { gen, stage: Full, result })
3. user_event(Decoded { gen, stage, result }):
     - gen != текущего → отбрасываем (юзер листнул дальше)
     - Ok(img): upload_texture + обновление вида (см. ниже)
     - Err(full): preview показан → остаётся preview + warn; иначе оверлей
```

```rust
pub enum Stage { Preview, Full }

pub enum UserEvent {
    Decoded {
        generation: u64,
        stage: Stage,
        result: std::result::Result<DecodedImage, String>,
    },
}
```

`DecodedImage { rgba, width, height }` — без изменений с v0.1.

### Подмена вида preview→full

Full обычно крупнее превью, но того же соотношения сторон. Чтобы экран не «прыгнул»:

- **первый кадр генерации** (preview, либо full если превью нет) → инициализация вида:
  `set_min_zoom(fit)`, `set_zoom_immediate(fit)`, `set_fit(true)`, `set_pan(0)` (как v0.1);
- **full поверх показанного preview** → пересчёт масштаба по соотношению размеров:
  `zoom *= preview_dim / full_dim`, затем `set_min_zoom(fit(full))`, `clamp_pan`.

Формула единообразна: при `is_fit` она даёт ровно `fit(full)`, при ручном зуме сохраняет
экранный размер. Инкапсулируется в тестируемый метод `ViewTransform`
(`rescale_for_new_image(win, old_img, new_img)` или аналог). `is_fit` не меняется.

App отслеживает «инициализирован ли вид для текущей генерации» (`Option<u64>`), чтобы отличить
первый кадр от подмены.

## RAW develop (rawler)

- **Embedded preview** (`decode_preview`): достаём встроенный JPEG-preview из RAW
  (есть в большинстве RAF/NEF/ARW/CR2), декодируем через `image` → `DecodedImage`.
  Превью нет → `None`.
- **Полный develop** (`decode_full`), базовый: rawler → `RawImage` (сенсор) → black/white
  levels → баланс белого (коэффициенты из метаданных) → демозаик → цветовая матрица
  cam→sRGB → sRGB-гамма → RGBA8. Опираемся на помощники `rawler`/`imgop`; точные сигнатуры
  API сверяются в фазе плана (Context7/docs). Цель <2с для 40 MP — основная стоимость в
  демозаике.
- full упал, а preview есть → остаёмся на preview.

Точный камерный цвет и film simulation — вне v0.2 (базовый develop «выглядит корректно»,
но не пиксель-в-пиксель как JPEG камеры; для этого есть embedded preview).

## HEIC (libheif-rs)

- `decode_full`: декод primary-изображения → interleaved RGBA → `DecodedImage`.
- `decode_preview` → `None` (одна стадия; полный декод HEIC и так быстрый).
- Требует нативный **libheif** (линковка) и **libclang** (bindgen на этапе сборки).

## Зависимости и сборка

`Cargo.toml` добавляет:

```toml
rawler     = "0.6"     # pure-Rust RAW (версия сверяется/закрепляется в плане)
libheif-rs = "0.18"    # HEIC/HEIF (FFI)
```

`image` уже есть с v0.1 (используется и для декода embedded JPEG-preview).

**libheif-rs на Windows — главный риск фазы.** До сборки нужно установить:

- **vcpkg** + `vcpkg install libheif` (тянет libde265/x265);
- **CMake**;
- **LLVM/Clang** → переменная окружения `LIBCLANG_PATH` для bindgen.

Митигация: **первая задача плана — smoke-сборка «hello libheif»**: крошечный бинарь, который
только линкуется с libheif и декодит один файл. Если окружение/линковка не заводятся — это
выясняется сразу, до реализации декодера. Версии крейтов закрепляются на проверенных.
Инструкция по окружению фиксируется в `ROADMAP.md` (раздел «Установленное окружение») и/или
`README`.

## Обработка ошибок (дополнения к v0.1)

Принцип прежний: один плохой файл не роняет приложение.

| Ситуация | Поведение |
|---|---|
| Превью есть, full упал | Остаётся превью на экране + `warn` |
| Нет превью, full упал | Прошлая текстура + текст-оверлей «Не удалось открыть файл» |
| RAW без embedded preview | Сразу full (без мгновенного превью) |
| Битый RAW/HEIC | `warn`, навигация продолжает работать |
| Паника в rawler/libheif на фоне | Ловится в rayon-задаче, уходит как `Err` через тот же `UserEvent` |

## Тестирование

GUI/wgpu-цикл юнит-тестами не покрываем. Чистая логика — по TDD:

- **роутер** (`decoder`): `supported()`/`decoder_for()` выбирают верный декодер
  (`raf`→Raw, `heic`→Heic, `jpg`→Standard, `txt`→None);
- **`view.rs`**: метод подмены preview→full сохраняет экранный размер (кейсы fit и ручной
  зум), обновляет `min_zoom`, клампит pan;
- **`decoder`**: `StandardDecoder::decode_preview` → `None`.

**Декод RAW/HEIC.** Фикстуры реальные и тяжёлые. Стратегия: положить по одному компактному
файлу (маленький DNG и маленький HEIC) в `tests/fixtures`, если умещаются в разумный размер;
иначе тесты декода помечаем `#[ignore]` (запуск вручную при наличии фикстур). Проверяем
размеры и длину RGBA-буфера. Решение по фикстурам принимается в плане после проверки размеров.

### Чек-лист ручной приёмки v0.2

1. Открытие RAF/NEF/ARW/CR2: превью появляется быстро (<200 мс), затем бесшовно сменяется
   полным develop без «прыжка» картинки.
2. Открытие HEIC показывает изображение.
3. Навигация стрелками по смешанной папке (JPEG+RAW+HEIC) в natord-порядке.
4. Зум/pan (из v0.1) работают одинаково для всех форматов.
5. Битый RAW/HEIC не роняет приложение (лог `warn`, навигация работает).
6. RAW без embedded preview открывается (сразу full).
7. Ресайз окна сохраняет fit для всех форматов.
