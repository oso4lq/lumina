# Lumina — v0.5b: наблюдатель папки (folder watcher) — дизайн

**Дата:** 2026-06-17
**Статус:** дизайн утверждён, готов к плану

## Проблема

`FolderCatalog::open` читает содержимое папки **один раз** при открытии файла и держит статический
отсортированный список. Если во время просмотра в папку добавили/удалили/переименовали файлы
(скопировали новую партию, удалили кадр, отработал экспорт Lightroom), Lumina об этом не знает:
карусель, счётчики `can_prev/can_next`, индексы — устаревают до следующего ручного переоткрытия.

## Решение

Наблюдать за открытой папкой через крейт `notify` (+ дебаунс), и при изменениях **пересобирать
каталог**, сохраняя текущий файл по пути. Кросс-поточные события воркера notify заводятся в
winit-event-loop через существующий `EventLoopProxy` (как декод-результаты).

### Поток данных

1. `open_file(path)` (после построения каталога) запускает наблюдатель за **родительской папкой**
   файла (нерекурсивно). Хэндл наблюдателя хранится в `App` (живёт, пока открыта папка; пересоздаётся
   при смене папки).
2. Воркер notify (свой поток) на событие ФС шлёт `UserEvent::FolderChanged` в event loop. События
   **дебаунсятся** (пакетируются с задержкой ~300 мс), чтобы взрыв событий при копировании партии
   свёлся к одному пересбору.
3. На `FolderChanged` приложение пере-сканирует папку и вызывает чистую логику согласования каталога
   (`FolderCatalog::refresh`), которая:
   - заново фильтрует/сортирует файлы (как `open`);
   - сохраняет «текущий» по **пути** (а не по индексу): если файл ещё существует — курсор на него;
     если исчез — на ближайший валидный индекс (тот же индекс, иначе последний);
   - сообщает, изменился ли набор/индекс.
4. Если набор изменился — обновить производные UI-данные (`thumb_count`, `active_index`,
   `can_prev/can_next`, `raw_flags`, `badge_labels`, `thumb_aspects`) и сбросить состояния миниатюр
   (`ThumbnailStore::reset` + перерисовка → видимые миниатюры перезапросятся). Если текущий файл
   исчез — выполнить переход (`load_current`) на новый текущий.

### Где меняется

- **`src/catalog/mod.rs` (чистое ядро):** новый метод `refresh`. Сигнатура:
  ```rust
  /// Пере-сканировать папку (повторное чтение ФС), сохранив текущий файл по пути.
  /// Возвращает true, если список файлов или текущий индекс изменились.
  pub fn refresh(&mut self) -> std::io::Result<bool>
  ```
  Внутри — повтор чтения каталога папки (та же фильтрация/сортировка, что в `open`) и релокация
  курсора по сохранённому пути. Для тестируемости логика релокации выносится в чистую функцию
  `relocate(files: &[PathBuf], prev_current_path: &Path) -> usize` (без ФС).
- **`src/watcher.rs` (новый, тонкая обёртка):** `FolderWatcher` — запускает дебаунс-наблюдатель за
  папкой, прокидывает события в `EventLoopProxy<UserEvent>` как `FolderChanged`. Хранит хэндл
  дебаунсера (его `Drop` останавливает поток). Логики согласования здесь нет — только мост ФС→event loop.
- **`src/app.rs`:** поле `_watcher: Option<FolderWatcher>`; запуск в `open_file`; новый
  `UserEvent::FolderChanged`; обработчик, дёргающий `catalog.refresh()` и обновляющий UI.
- **`Cargo.toml`:** зависимости `notify` и `notify-debouncer-mini`.

### Выбор крейтов

- `notify` (v6/v8) — де-факто стандарт ФС-наблюдения; на Windows использует `ReadDirectoryChangesW`.
- `notify-debouncer-mini` — лёгкая обёртка дебаунса (батч событий за окно). Достаточно: нам не важен
  тип каждого события, важен факт «папка менялась» → один пересбор.

### Границы (осознанно)

- **Нерекурсивно:** наблюдаем только саму папку открытого файла (как и каталог — плоский список).
- **Текущий кадр не дёргаем зря:** если текущий файл цел и набор не изменил его позицию для показа,
  перерисовываем карусель/счётчики, но не перезагружаем изображение.
- **Сохранение аспектов:** при изменении набора `thumb_aspects` пере-инициализируется (как при смене
  папки) — индексы съехали, старые аспекты невалидны. Это сознательный компромисс простоты против
  инкрементального сдвига (YAGNI).
- **Внешнее редактирование текущего файла** (перезапись содержимого) отдельно не обрабатываем в v0.5b —
  notify пришлёт событие, пересбор не изменит список → no-op показа. Авто-перечитывание изменённого
  кадра — возможная последующая фича.

## Тестирование

Юнит (чистое ядро, без ФС):
- `relocate`: текущий файл существует → его новый индекс; удалён → тот же индекс или последний при
  выходе за границу; пустой список → 0.
- `FolderCatalog::refresh` — через тестовый конструктор (`from_files_for_test`) проверить смену
  набора и сохранение/релокацию курсора (рефакторить `refresh` так, чтобы ядро релокации не требовало ФС).

Интеграция/ручная приёмка: открыть папку → в проводнике добавить файл → в течение ~0.5 с он
появляется в карусели; удалить текущий → курсор переходит на соседний; счётчики/стрелки корректны.
