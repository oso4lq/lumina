# Дизайн v0.3a — движок UI-примитивов + frameless titlebar

Дата: 2026-06-15. Фаза: v0.3a (первая под-фаза v0.3 — UI).
Спека: `SPEC_lumina.md` §4 (UI), §10 (тема), §11 (DPI). Roadmap: `ROADMAP.md`.

## Контекст и решения

v0.3 (UI-слой) дробится на под-фазы: **v0.3a** — движок UI-примитивов + frameless
titlebar; **v0.3b** — divider + мета-панель + карусель + кнопки действий (отдельная
спека позже, поверх готового движка).

Принятые на брейншторме развилки:
- **Рендер UI** — самописные wgpu-примитивы (как задумано в спеке), не egui.
- **Рендер текста** — glyphon (cosmic-text + кэш-атлас глифов): кириллица, Unicode,
  sub-pixel из коробки.
- **Иконки** — шрифт Tabler Icons (.ttf) через тот же glyphon (спека ссылается на
  `ti-maximize`, `ti-info-circle`); без отдельного атласа-пайплайна.
- **Frameless-окно** — полноценный нативный фрейм через `WM_NCCALCSIZE` /
  `WM_NCHITTEST` (нативные resize / Aero Snap / тень / скруглённые углы Win11),
  не базовый winit-drag.

## Цель и границы

Построить фундаментальный слой рендера UI поверх wgpu и кастомное frameless-окно с
нативным фреймом, плюс titlebar (кнопки управления окном + заголовок по центру).

**Не входит в v0.3a:** карусель, divider, bottom bar, мета-панель, кнопки
действий, EXIF popup — это v0.3b/v0.4.

**Нюанс зависимости (EXIF).** По спеке центр titlebar — `имя · модель_камеры_из_EXIF ·
Lumina`. Чтение EXIF — это v0.4. Поэтому в v0.3a центр показывает **`имя_файла ·
Lumina`**; модель камеры подключается в v0.4, когда появится EXIF. Это сознательная
заглушка, а не недоделка.

## Архитектура модулей

Соблюдаем принцип проекта: «чистое ядро» тестируется юнитами, GPU/winit/FFI — тонкие
обёртки без логики.

### Чистое ядро (юнит-тестируемое, без GPU/winit)

- `src/ui/layout.rs` — по размеру окна и `scale_factor` считает прямоугольники
  titlebar: кнопки min/max/close, область заголовка, drag-регион. Возвращает `UiLayout`.
- `src/ui/theme.rs` — `ThemePalette` (цвета §10) + константы отступов. Чистые данные.
- `src/ui/scene.rs` — по `(UiState, UiLayout, ThemePalette)` строит список draw-команд
  (rect'ы, текстовые раны, иконки). Тестируется: при заданном state — ожидаемые команды
  в ожидаемых позициях.
- `src/ui/hit.rs` — чистый hit-test: курсор + layout → регион
  (`Close` / `Minimize` / `Maximize` / `CaptionDrag` / `ResizeEdge(..)` / `None`).

### Тонкие обёртки (GPU/FFI, без логики)

- `src/renderer/ui_pipeline.rs` — `UiPipeline`: батч скруглённых прямоугольников через
  SDF в `assets/shaders/ui.wgsl`; instance-буфер `{pos, size, color, corner_radius}`.
- `src/renderer/text.rs` — обёртка над glyphon (`FontSystem`, `SwashCache`, `TextAtlas`,
  `TextRenderer`); рисует текстовые раны из сцены. Иконки — глифы Tabler через тот же путь.
- `src/platform/windows.rs` — нативный frameless-фрейм (см. ниже).

## Композиция кадра

Один surface-frame, один command encoder, порядок:
1. **Blit фото** (существующий `BlitPipeline`) в **viewer-регион** (ниже titlebar), а не
   на всё окно — через scissor/viewport.
2. **UiPipeline** — фон titlebar, кнопки, hover-подложки.
3. **Текст/иконки** (glyphon) — заголовок, глифы кнопок.

**Следствие для view (единственная правка кода v0.1).** Сейчас фото занимает всё окно.
`ViewTransform::matrix` и fit/zoom должны считаться относительно viewer-региона, а не
всего окна. Пробрасываем viewer-rect в view; покрываем юнит-тестом (матрица против
суб-региона).

## Нативный frameless (`src/platform/windows.rs`)

Берём HWND из winit (`raw-window-handle`), субклассируем wndproc (`SetWindowSubclass`,
цепочка через `DefSubclassProc` — winit-проц остаётся в цепочке). Обрабатываем
минимальный набор (рецепт Windows Terminal / tao):

- **WM_NCCALCSIZE** — расширяем client area на всё окно (убираем caption), сохраняя
  `WS_THICKFRAME` для resize/snap. Тень — `DwmExtendFrameIntoClientArea` на 1px.
- **WM_NCHITTEST** — caption-drag → `HTCAPTION` (даёт нативные drag,
  double-click-maximize, snap, shake бесплатно), края → `HTLEFT/HTTOP/...`. Сопоставление
  курсор→регион берём из чистого `hit.rs`; Win32-слой лишь мапит результат в `HT*`-коды.

Кнопки min/max/close обрабатываются в client area через winit-события
(`set_minimized` / `set_maximized` / exit).

**Риск.** Это самый рискованный кусок (FFI + чужой wndproc на MSVC) — в плане выделяется
отдельная задача-спайк до основной реализации. **Деградация:** если субклассинг не удался —
лог-предупреждение и откат на нативные декорации; приложение всё равно работает.

## Поток состояния

`App` держит `UiState` (hover/pressed кнопок, текст заголовка = имя текущего файла, флаг
maximized). `window_event` (cursor move / mouse input) → обновить `UiState` + hit-регион →
`request_redraw`. На redraw: `layout → scene → renderer` (UiPipeline + текст).

## Масштаб и тема

Все размеры в логических px × `scale_factor`. `ScaleFactorChanged` → пересчёт layout
(§11). Тема — тёмная `ThemePalette` зашита (System/Light — v0.6), но struct и
параметризация шейдеров делаем сразу, без хардкода hex в рендерере (§10).

## Обработка ошибок

- Win32-вызовы: при сбое субклассинга — лог + откат на нативные декорации.
- glyphon/UiPipeline-инициализация: ошибки оборачиваются в `LuminaError` (новый вариант
  при необходимости), фатальны на старте рендера — как и текущая инициализация GPU.

## Тестирование

Юнитами (чистое ядро):
- `layout` — rect'ы кнопок для разных размеров окна и `scale_factor`.
- `hit` — курсор→регион, включая края resize и caption.
- `theme` — дефолтная палитра.
- `scene` — `(state)` → ожидаемый набор draw-команд и их позиции.
- `view` — матрица против viewer-суб-региона (новый тест).

GPU/Win32 — ручная приёмка. Существующие 29 тестов остаются зелёными.

## Критерии приёмки

- Frameless-окно с нативными resize / Aero Snap / тенью / скруглёнными углами.
- Titlebar: кнопки min/max/close с hover, drag за caption, double-click → maximize, snap.
- Центр titlebar: `имя_файла · Lumina`.
- Фото в viewer-регионе под titlebar; zoom/pan/fit корректны относительно него.
- Тёмная `ThemePalette` через `ui.wgsl`, без хардкоженных hex в рендерере.
- Все тесты зелёные (существующие + новые юниты ядра UI).

## Новые зависимости

- `glyphon` — рендер текста поверх wgpu.
- `windows` (crate) — Win32 FFI для нативного фрейма (`WM_NCCALCSIZE`, `WM_NCHITTEST`,
  `Dwm*`, субклассинг).
- `raw-window-handle` — получение HWND из winit (вероятно уже в дереве через winit/wgpu).
- Tabler Icons `.ttf` — ассет шрифта иконок.
